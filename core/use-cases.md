# Breadcrumbs Use Cases

This document describes typical scenarios where Breadcrumbs can be used to facilitate reviews and collaboration.

## Documentation Reviews

Instead of using a separate review tool, comments can be placed directly next to the relevant content.

---

## Human ↔ AI Collaboration

An AI agent generates a document.

A human reviewer leaves feedback using Breadcrumbs.

The AI agent updates the document and resolves comments.

The review discussion remains attached to the document throughout the process.

---

## Agent ↔ Agent Collaboration

Multiple agents can collaborate through the same document.

For example:

1. Planning Agent creates a proposal.
2. Review Agent identifies concerns.
3. Implementation Agent addresses comments.
4. Human Reviewer approves the final result.

Because Breadcrumbs are machine-readable, tools can automate much of this workflow.

---

## Portable Reviews

Unlike comments tied to a specific platform, Breadcrumbs remain inside the document.

The review context remains available when the document is:

* Shared as a file
* Moved between repositories
* Processed by AI tools
* Exported to other systems
* Archived for future reference
